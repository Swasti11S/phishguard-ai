HOW IT WORKS
01
Feature Extraction
When you submit a URL, 8 numerical features are extracted: URL length, IP address presence, dot count, @ symbol, HTTPS usage, suspicious keyword count, hyphen count, and subdomain depth.
02
Logistic Regression
Features are fed into a Logistic Regression model. It computes a weighted sum and passes it through a sigmoid function to output a 0–1 phishing probability.
03
Training Data
The model was trained on 10,000 URLs — 5,000 safe (real websites) and 5,000 phishing (IP-based, lookalike domains, @ redirects, bad TLDs, excessive subdomains).
04
Verdict + Confidence
If phishing probability exceeds 50%, the URL is flagged. Confidence % reflects how strongly the model leans toward either class based on the feature vector.
FEATURE REFERENCE
URL LengthLong URLs hide real domains. Anything over 75 chars is suspicious.
Has IP AddressRaw IPs like 192.168.1.1 bypass domain reputation filters.
Dot CountMultiple dots = excessive subdomains — a classic phishing trick.
Has @ SymbolBrowsers ignore everything before @ — used to disguise destinations.
Uses HTTPSLegit sites use HTTPS. HTTP-only is a strong phishing indicator.
Suspicious KeywordsWords like login, verify, free, prize are abused in phishing URLs.
Hyphen CountHyphens mimic brands — paypal-secure-login.tk is a classic example.
Subdomain Countpaypal.secure.evil.com — deep nesting hides the real domain (evil.com).
